Building My NPLQ Revision App: First Steps into React, Firebase, and Version Control

Over the course of July, I taught three RLSS NPLQ courses, and above all, the one thing that I took away with me was the severe lack of interactive material available to the candidates. Their only option was to read from the course manual, and I'm sure that after spending eight hours with me that is the last thing they all wanted to be doing in the evenings. I decided I wanted to change that, and over the past few weeks I've been working on building a full-stack revision app for the NPLQ qualification. What started as a relatively simple idea - creating a flashcard bank - has now grown into a fully functional web app with user authentication, quizzes, flashacards and an admin dashboard.

Choosing the Stack

This was my first time using React and I quickly saw why it is so popular. The component structure made it straightforward to split the app into smaller parts such as the login form, dashboard and flashcard view, and then put them back together into a single application.

For the backend, I integrated Firebase as it offered a simple and effective way to manage user authentication and data storage. Firestore allowed me to create databases for my flashcards, quizzes and user progress, and Firebase Authentication ensured each learner had an individual login, while also being able to distinguish between student and admin roles.

Hosting and Deployment

Once I had the app running locally, I chose Vercel for deployment, due to its smooth GitHub integration. This was the first time I had experienced live deployment, and it was incredibly satisfying to see my app go from local development to a live environment with just a few clicks.

Styling and User Experience

I styled the app using CSS, beginning from an incredibly simple interface and gradually adding more complex styles as I went along. I focused on making the user experience as intuitive as possible, ensuring consistency across the design system. I picked a soft blue and red colour scheme and added layouts with flexbox, refined box shadows, card styles and progress bars until the whole app felt polished and easy to use.

Learning New Tools

This was also my first use of GitHub for version control. At the start, I simply pushed changes as backups, but also began working with commits and rollbacks to keep my progress safe and allow me to feel comfortable taking risks, knowing I can always go back if needed.

Features

The app includes both flashcards and quizzes, organised by course section to make revision more structured. Quizzes feature a mix of question types, including multiple choice, fill in the blank, drag and drop ordering, and matching pairs. Learners can track their progress with a timer that records time spent revising, along with counts of completed flashcards and quizzes. The dashboard gives a simple overview of progress, while separate user and admin logins keep access secure. Looking ahead, I am working on team functionality and leaderboards so course mates can compare their revision and compete together.

Reflections

This Web App represents many firsts for me:

  • My first experience with full-stack development
  • My first time using React for the frontend
  • My first time integrating Firebase for user authentication
  • My first live deployment using Vercel
  • My first project using GitHub for version control
What I thought would be a simple afternoon coding a side project became a deep dive into frontend frameworks, backend integration and deployment strategies. I've learnt how to design and build a complete web application from scratch from just an idea, and the confidence and curiosity I have gained from it will undoubtedly influence my future projects.
